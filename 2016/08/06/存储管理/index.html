存储管理

Posted by srtianxia on 2016-08-06

存储管理的概念

存储器用于存放程序（指令）、操作数（数据）以及操作结果。计算机系统中，存储器一般分为主存储器和辅助存储器两大类。CPU可以直接访问主存储器中的指令和数据，但不能直接访问辅助存储器。在I/O控制系统管理下，辅助存储器与主存储器之间可以进行信息传递。

  • 主存储器简称主存，或称为内存。主存可分为系统区和用户区两个区域。
  • 存储管理是对主存中的用户区进行管理，其目的是尽可能方便用户和提高主存空间的利用率，使主存在成本、速度和规模之间获得较好的权衡。

在现代计算机系统中，存储部件通常采用层次结构来组织，以便在成本、速度和规模等因素中获得较好的性价比。

现代通用计算机，存储层次至少应具有三级

  • 最高层为CPU寄存器
  • 中间层为主存
  • 最底层为辅存

现代高级计算机系统可以分为下面几层

上面四个均属于操作系统存储管理的管辖范围，掉电后它们存储的信息不再存在。下面两个属于设备管理的管辖范围，可以长期存储。磁盘缓存本身并不是一种实际存在的存储介质，它依托于固定磁盘，提供对主存储器存储空间的扩充。

内存管理的主要任务

  • 合理的建立用户程序与内存空间的对应关系
  • 为各道程序分配内存空间
  • 运行完成后再予以回收
  • 始终保证系统程序和各用户程序安全

物理地址和逻辑地址

  • 主存储器的存储单元以字节（每个字节为8位二进制）为单位编址，每个存储单元都有一个地址与其相对应。假定主存储器就有n个字节的存储空间，其地址编号为0，1,2…n-1。这些地址称为主存储器的物理地址（绝对地址），由物理地址所对应的主存空间称为物理地址空间。
  • 多道程序设计系统中，主存中同时存放了多个用户作业。每次调入运行时，操作系统将根据主存的使用情况为用户分配主存空间。为了方便程序编制，每个用户可以认为自己作业的程序和数据存放在一组以“0”为地址开始的连续空间中。用户程序中所使用的地址称为逻辑地址，由逻辑地址对应的存储空间称为逻辑地址空间。

程序的装入

将一个用户源程序装入主存中并执行，通常需要经过以下几个步骤：首先由编译程序（Compiler）将用户源代码编译成若干目标模块（Object Module）；然后链接，由链接程序（Linker）将编译后形成的一组目标模块以及它们所需要的库函数链接在一起，形成一个完整的装入模块（Load Module）；最后装入，由装入程序（Loader）将装入模块装入主存。

将一个装入模块装入主存，可以有绝对装入方式、静态重定位装入方式和动态重定位装入方式。

重定位：当一个逻辑地址空间的程序装入到物理地址时，由于两个空间不一致，需要进行地址映射。

  • 绝对装入方式
    如果在编译时知道程序驻留在主存的具体位置，则编译程序将产生物理地址的目标代码。绝对装入程序按照装入模块中的地址，将程序和数据装入主存。模块装入后由于程序中的逻辑地址与实际主存的地址完全相同，所以不需要对程序和数据的地址进行修改。
    绝对装入方式只能将目标模块装入主存储器事先指定的固定位置，它只适用于单道程序环境。

  • 静态重定位装入方式
    在装入作业时，把该作业中的指令地址和数据地址一次性全部转换为物理地址，这样在作业执行过程中无需再进行地址转换工作，这种地址转换方式称为静态重定位，这个工作通常由装配程序完成。

    优点：容易实现，无需硬件支持，它要求程序本身是可重定位的，即对那些要修改的地址部分具有某种标示，地址重定位由专门设计的程序来完成。
    缺点：程序重定位后不可移动，因而不能重新分配内存，不利于内存的有效利用。

  • 动态重定位装入方式
    在装入作业时，装入程序直接把作业装入到所分配的主存空间中。在作业执行过程中，随着每条指令或数据的访问，由硬件地址转换机制自动地将指令中的逻辑地址转换成对应的物理地址。这种地址转换的方式是在作业执行中动态的完成的，故称为动态重定位，通常采用的办法是利用一个重定位寄存器，对每一个有效地址都要加上重定位寄存器中的内容，以形成绝对地址。

    采用动态重定位时，由于装入主存的作业扔保持逻辑地址，所以必要时可改变作业在主存储器中的存放位置。
    优点：程序在内存中的搬移不会对程序的正确执行造成影响，使内存得以被充分利用
    缺点：需要附加的硬件支持，实现存储管理的软件比较复杂

虚拟存储器概念的引入

就是人人们经常说的虚拟内存，是使用一部分外存当做内存来使用，使内存扩大，是对用户透明的

分区式存储管理

  • 单一连续分配
  • 分区分配
    • 固定分配
    • 动态分配
    • 动态重定位分区

单一连续分配

主要用于单用户单任务操作系统（如DOS）
实施方案

  • 内存分配
    整个内存划分为系统区和用户区
  • 地址映射
    物理地址 = 用户区基地址 + 逻辑地址
  • 内存保护
    基址寄存器、界限寄存器、越界中断信号


单一连续存储管理每次只允许一个作业装入主存储器，因此可以不必考虑作业在主存中移动的问题，所以当作业被装入主存时，系统一般采用静态重定位进行地址转换，当然动态也可以。

单一连续存储管理方式下的存储保护比较简单，即判断物理地址>=界限地址，并且物理地址<=主存最大地址，条件成立则执行，否则形成地址越界程序性中断事件。当采用静态重定位装入方式时，又装入程序检查其物理地址是否超过界限地址，若是，则可以装入，否则产生地址错误程序不能装入。这样一个被装入的作业，总能保证在用户区中执行，避免破坏系统区中的信息，达到存储保护的目的

固定式分区分配

应多道程序的存储管理而产生，这里的固定指的是分区数目和分区大小固定，固定分区存储管理预先把主存储器中的用户区分割成若干个连续区域，每一个连续区域称为一个分区，每个分区的大小可以相同也可以不同。分割完成后，每个分区的大小和分区的数量就不变了。每个分区只能装入一个作业，不允许一个作业跨分区存储

  • 内存分配/回收
    分区说明表（由系统建立）
  • 地址映射
    物理地址 = 分区地址 + 逻辑地址
  • 内存保护
    分区始地址、逻辑地址与所给分区大小相比较（见上方分区说明表）

缺点
内存空间的利用率低，有较多的碎片，下图只有25%的利用率

关于固定式分区优化可以采用多个作业队列的固定分区法，使用多个队列（按照进程大小分队列：大进程队列，小进程队列）系统分区时先预先分区为大小不同的分区，之后每个队列在大小不同的分区处等待调度，可以更好的利用空间

动态分区分配

可变式分区分配策略，在进程装入和处理过程中建立分区，并要使分区的容量能正好适应进程的大小，换句话说就是让下面两个量一一对应。
内存分区数目 -> 进程数目
分区的大小 -> 各个进程的大小

可变分区的数据结构

采用可变分区存储管理方式管理存储空间时，主存中已占用分区和空闲区的数目大小都是可变的，为了实现可变分区和已分配分区的情况，为系统空间分配提供依据。常用的数据结构有以下两种形式。

  • 分区表
    系统设置空闲分区表和已分配分区表，用来描述空闲分区和已分配分区的情况，为系统空间分配提供依据。
  • 分区链
    为了实现对空闲分区的分配和链接，在每个分区的初始部分，设置一些用于控制分区分配的信息，以及用于链接前一个分区的前向指针；在分区尾部则设置一后向指针，通过前后向链接指针，可以将所有的空闲分区连接成一个双向链。为了检索方便，在分区尾部重复设置状态位和分区大小表目。

可变分区分配算法

将一个作业装入主存，须按照一定的分配算法，从空闲分区表或空闲分区链中选出一个分区分配给该作业。可变分区存储操作管理常用的空间分配算法有：

  • 首次适应算法（FF，Frist Fit）
    空闲分区链以地址递增的顺序链接，分配时从链首开始查找，找到第一个大小可以满足的分区为止。
    经过若干次作业的装入与撤离后，有可能把较大的主存空间分割成若干小的、不连续的新空闲分区，这些空闲分区的长度可能比较小，不能满足主存再次分配的需要，从而使主存空间的利用率大大降低这些空闲分区称为碎片.
  • 循环首次适应算法
    为了改变首次适应算法每次从链首开始查寻造成的缺陷，可以增加一个起始查寻指针，指向下一次开始查寻时的起始分区，在查寻过程中，该指针向后移动，当移动到最后一个空闲分区后，重新回到链首。
    这种方法可以使内存区分配比较平均，减少查寻次数，但又造成了缺少大空闲分区，使得大进程无法进入。(原因 ？？？？)
  • 最佳适应算法
    空闲分区按照分区容量递增的方式形成，分配时从链首开始查找，平均查找次数为分区数的一半，也就是说它是最佳适应的。
    这种方法所造成的剩余分区又肯定是相对最小的，通常难以利用，甚至我们可以说，每次分配一个分区都会造成一个无法再利用的碎片，而这些碎片的总容量可能是比较大的。
  • 最差适应算法
    并非最差，空闲分区链按照容量递减的方式形成，分配时从链首开始，若链首分区大小不满足，则可以肯定不纯在能够满足要求的分区；否则对链首分区进行划分，剩余空间成为碎片的可能性肯定是最小的。
    具有查找速度快分区碎片少的优点，但又会产生缺少大分区的缺点。

动态分区的回收

装入的作业执行结束后，它所占据的分区将被回收，回收后的空闲分区登记在空闲分区表中，用来装入新的作业。回收空间时应检查是否存在与回收区相邻的空闲分区，如果有，则将其合并为一个新的空闲分区进行登记管理。

动态重定位分区分配

关于在动态分区分配方法中，碎片问题可以采用紧凑的方式，使分散的空闲分区集中起来以容纳新的作业。
紧凑：

动态重定位分区分配 = 动态分配 + 紧凑

分区分配方案的评价

地址映射 -> 物理地址 = 分区始地址 + 逻辑地址

固定分区分配 -> 基址寄存器
动态分区分配 -> 基址寄存器
动态重定位分区分配 -> 重定位寄存器

内存保护 ->基址寄存器and限长寄存器or下限寄存器and上限寄存器

分区分配方案的主要优点
  • 多道程序下的内存共享，改善了系统吞吐量，在内存利用率方面，从固定式分区到动态分区，再到动态重定位分区依次提高。
  • 相对于后面介绍的存储管理方式，为实现分区分配所使用的数据结构占用的存储容量相对较少，算法也相对简单。
  • 实现存储保护的措施也比较简单。
    分区分配方案的主要缺点
  • 内存仍不能充分利用，处了动态重定位式分区法外，都存在着严重的碎片问题。
  • 不能实现对内存的扩充，因此进程的大小收到内存可用空间的限制。
  • 与单一连续分配一样，要求一个进程在执行之前必须全部装入内存，因此在内存中可能包含不会被使用的信息。
  • 紧凑提高了内存利用率，但是进程移动所产生的额外开销增加了CPU的负担。
  • 几个并行进程之间不能共享存入内存的单一信息副本（公用子程序，数据段等）
