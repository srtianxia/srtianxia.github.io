Android Looper Handler

Android消息处理机制

Posted by srtianxia on 2016-08-06

Looper

Looper的作用是使一个线程变为looper线程，即消息循环的线程，内部实现简单来说就是维护了一个阻塞队列（实际上MessageQueue是一个单链表）并实现了一个死循环，没有消息的时候就阻塞等待消息的到来。

每一个Thread只有一个Looper，由ThreadLocal存储，ThreadLocal是一个线程本地存储变量，内部维护了一个map，用当前线程作为key来存储一定的value，这些value是线程间私有的，也就是其他线程不能访问到当前线程对应的value，这样就保证了每一个线程只对应一个Looper对象，每一个Looper还维护着对应的线程mThread和 一个 消息队列mMessageQueue

让一个线程变为Looper线程只需要调用下面这两步

  • Looper.prepare()
    会先检查当前线程是否已经拥有了Looper对象（调用mThreadLocal.get()方法并检查返回值是否为空，在get()内部会调用Thread.currentThread()来获取当前线程并取得value也就是Looper对象）在这里会初始化MessageQueue (初始化看下文)和将mThread赋值为当前线程。
  • Looper.loop()
    loop()方法是一个死循环,就是从MessageQueue中取消息(Message)来交给Message的target来处理，这个target是一个handler对象会调用dispatchMessage()来处理消息，这个方法在下文的handler中会介绍，跳出循环的条件是MessageQueue为空，而让这MessageQueue为空的办法不是简单的消息取完了，请看下面对于MessageQueue的介绍。

MessageQueue

构造函数为包级私有，内部由Native方法nativeInit()实现，在外部除了调用Looper.prepare()方法，外部无法直接初始化。

在nativeInit()中，new了一个Native层的MessageQueue的对象，并将其地址保存在了Java层MessageQueue的成员mPtr 中 Android中有好多这样的实现，一个类在Java层与Native层都有实现。

  • enqueueMessage()
    向消息队列中插入Message
  • next()
    是一个无限循环的方法，如果消息队列中没有消息，next()方法就会一直阻塞在这里，当有消息到来时，next()会返回这条消息，并将其从消息队列中移除，所以对于上文的loop()方法，消息取完了消息队列也处于阻塞状态，置空方法只有调用quit或者quitSafely方法来通知消息队列退出。

Handler

发送消息

handler的post() 、postDelayed()、sendMessage()等方法最终都是调用sendMessageAtTime 方法。

这个方法由获取到消息队列（消息队列mQueue是从与这个handler关联的Looper获得的）然后调用enqueueMessage()方法。

方法enqueueMessage()将message的target设置为当前的handler，然后调用MessageQueue的enqueueMessage(上文已经介绍)。

在调用queue.enqueueMessage之前先判断了mAsynchronous，从名字看是异步消息的意思，要明白Asychronous的作用，需要了解一个概念Barrier。

Barrier从名字上来看上是一个拦截器，在这个拦截器后面的消息暂时都无法执行，直到这个拦截器被移除，MessageQueue有一个函数叫enqueueSyncBarrier可以添加一个Barrier 。

在enqueueSyncBarrier中，先obtain一个Message，并设置了msg arg1 ＝ token，token仅是一个每次调用enqueueSyncBarrier时自增的int值，目的是每次调用enqueueSyncBarrier时返回唯一的token，这个Message同样需要设置执行时间，然后插入到消息队列，特殊的是这个Message没有设置target，即msg target ＝ null。

Message

在Java层，Message类表示一个消息对象，要发送消息首先就要先获得一个消息对象，Message类的构造函数是public的，但是不建议直接new Message，Message内部保存了一个缓存的消息池，我们可以用obtain()从缓存池获得一个消息，Message使用完后系统会调用recycle()回收，如果自己new很多Message，每次使用完后系统放入缓存池，会占用很多内存的。

Message类中有一个同类型的next，还还维护了一个static Message 类型的sPool作为缓存链表。
